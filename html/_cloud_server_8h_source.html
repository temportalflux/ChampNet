340 // Key subscription
341 //
342 // A given system can subscribe to one or more keys.
343 // The subscription can be further be defined as only subscribing to keys uploaded by or changed by a given system.
344 // It is possible to subscribe to keys not yet uploaded, or uploaded to another system
345 //
346 // Operations:
347 //
348 // 1. SubscribeToKey() - Get() operation with subscription
349 // A. Add to key subscription list for the client, which contains a keyId / specificUploaderList pair
350 // B. Send to remote servers that for this key, they should send us updates
351 // C. (Done, get operation returns current values)
352 //
353 // 2. UpdateData() - Post() operation
354 // A. Find all subscribers to this data, for the uploading system.
355 // B. Send them the uploaded data
356 // C. Find all servers that subscribe to this data
357 // D. Send them the uploaded data
358 //
359 // 3. DeleteData() - Release() operation
360 // A. Find all subscribers to this data, for the deleting system.
361 // B. Inform them of the deletion
362 // C. Find all servers that subscribe to this data
363 // D. Inform them of the deletion
364 //
365 // 4. Unsubscribe()
366 // A. Find this subscriber, and remove their subscription
367 // B. If no one else is subscribing to this key for any system, notify remote servers we no longer need subscription updates
368 //
369 // Internal operations:
370 //
371 // 1. Find if any connected client has subscribed to a given key
372 // A. This is used add and remove our subscription for this key to remote servers
373 //
374 // 2. For a given key and updating address, find all connected clients that care
375 // A. First find connected clients that have subscribed to this key, regardless of address
376 // B. Then find connected clients that have subscribed to this key for this particular address
377 //
378 // 3. Find all remote servers that have subscribed to a given key
379 // A. This is so when the key is updated or deleted, we know who to send it to
380 //
381 // 4. For a given client (such as on disconnect), remove all records of their subscriptions
