DataStructures::List<uint32_t> RakNet::CloudQueryResult::resultKeyIndices

If there were multiple keys in CloudQuery::keys, then each key is processed in order and the result concatenated to rowsReturned The starting index of each query is written to resultKeyIndices For example, if CloudQuery::keys had 4 keys, returning 3 rows, 0, rows, 5 rows, and 12 rows then resultKeyIndices would be 0, 3, 3, 8
